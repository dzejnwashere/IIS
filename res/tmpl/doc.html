Implementace

Stručná dokumentace k implementaci, která popisuje, které části projektu (např. PHP skripty) implementují jednotlivé případy použití.

AUtorizace pomocí JWT tokenu ukládajícího ID přihlášeného uživatele, vše ostatní řešeno na backendu. Hesla ukládaná pomocí bcrypt, většina funkcí na práci s přihlášeným uživatelem žije v kódu v /auth

Veškerá interakce s databází je pomocí modulu /db v kódu, který zajištuje přímé vytahování a zpracování dat z DB

Ostatní práce (s požadavky, prezentace uživatelům, API) je rozmístěna mezi main.go, lines.go a rides.go

Instalace

  • Program vyžaduje nainstalovaný toolchain go a databázi mysql
  • Inicializaci databáze si program zajistí interně sám při prvním spuštění, ukázková data do databáze vloží po přihlášení administrátorským účtem (admin/admin) a návštěvě adresy "/demo"
  • Program se spouští na lokálním portu 53714
  • na serveru: export DBSTRING=<Your DB connection string here>, e.g. export DBSTRING=xplagiat0b:tajneheslo@unix(/var/run/mysql/mysql.sock)/iisdbname
  • V adresáři projektu: go build
  • V adresáři projektu: ./IIS

Známé problémy

Zde popište, které body zadání nejsou implementovány a z jakého důvodu. Např. „Z časových důvodů nebyla implementována správa uživatelů.” Pomůžete tím zrychlit hodnocení, když neimplementované funkce nebudeme muset dlouze hledat.

Dosť sme si stretlili do nohy tým, že sme na back-end použili jazyk GO. Bola to naša prvá skúsenosť s týmto jazykom a popravde nie veľmi pozitívna. Keď sme si uvedomili, ako sme si tým projekt skomplikovali, bolo už neskoro na prerobenie back-endu.
